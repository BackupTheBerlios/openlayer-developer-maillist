Well it really depends on the restrictions you want to impose on the end user? 
If you want a generic or near to the public domain license, I'd look into png 
or zlib licenses. 
LGPL, however has an advantage over GPL in that you can link against the 
library and not have to LGPL your own code. However in tobings case I think 
he includes all the source to every library in his projects, which is what 
would cause his code to be infected. I didn't even know, honestly, that it 
was licensed under LGPL, there is no mention of the LGPL license in the 
openlayer sources. You should throw in a LICENSE file in the root directory, 
that way there are no confusions or misunderstandings.
