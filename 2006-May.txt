I've finally gotten around to start inclusion of FreeType within openlayer.
I've created a new class called Glyph and have added Glyph.{cpp/hpp} to the 
lib. So far I've set it up so that it provides an alternative to GlyphKeepers 
routines within TextRenderer that way it doesn't break the API (although much 
of the routines are incomplete). As the Glyph class matures, it will 
eventually become more than capable of replacing GlyphKeeper itself which at 
that point we can remove any reference to GK and drop the compatible routines 
in Glyph and finally the GK dependency.
I will continue to work on it when I have time.
Fladimir I encourage you to continue to use GlyphKeeper's library if you need 
to improve or tweak TextRenderer in the meantime.
As such, the library will compile if you decide to enable internal TTF, but be 
warned that it's mostly guaranteed not to do anything and might even crash 
your applications. If you are still interested in seeing it or testing it 
out, you must enable the developer flag when compiling the library. At which 
point it will prompt you for inclusion of the new support during the 
configuration prompts.
Thus on linux/macosx/msys you can do (make sure it is clean first):
$ ./cbuild.c developer

or

windows cmd.exe:
c:\buildme.bat developer
